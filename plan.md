# Implementation Plan — Agentic Marketing (NorthStar Health)

This plan aligns the `narrative.md` / `context.md` scenario (NorthStar Health consumer
healthcare) with the proven technical patterns in `src-samples/` and `scripts-samples/`
(the agentic-banking ecosystem). We **keep the technical flow** of the samples and only
**change the business domain and data** (banking → consumer‑healthcare marketing).

> Golden rule: application code → `src/`, deployment/automation → `scripts/`, all Python
> packages consolidated in the root `requirements.txt`. Each new artefact is a
> business‑domain re‑skin of a named sample file listed under **Reference**.

## 2. Data & AI Search Indexes

The datasets already exist (generated by `scripts/generate_datasets.py`) and already
contain readable `description` fields:
- `data/products.json` — 50 NorthStar + competitor products, `description` present.
- `data/personas.json` — 90 personas (30 × 3 markets), narrative `description` present.
- `data/research_innovations.json` — innovations with narrative `description`.
- `data/market_sales_monthly.json` / `data/market_sales_annual.json` — sales history + forecast.

### 2.1 Three AI Search indexes

**Reference:** `scripts-samples/create_search_indexes.py` (schema, HNSW vector search,
semantic config, `_vector_field`, `_semantic` helpers, `SearchIndexClient`).

| Index (default name) | Key fields | Vector field(s) | Semantic content / keywords |
| --- | --- | --- | --- |
| `products` | `id`, `product_name`, `category`, `brand`, `market` (Collection), `price_tier`, `list_price`, `gross_margin`, `is_competitor`, `claims` (Collection), `launch_year`, `tags`, `description` | `description_vector` | content=`description`, keywords=`claims`,`tags` |
| `personas` | `id`, `name`, `market`, `archetype`, `age`, `gender`, `income_band`, `risk_tolerance`, `digital_maturity`, `interests` (Collection), `preferred_categories` (Collection), `preferred_channel`, `annual_spend`, `description` | `description_vector` | content=`description`, keywords=`interests` |
| `innovations` | `id`, `innovation_name`, `category`, `technology`, `concept`, `trends_addressed` (Collection), `estimated_cost`, `estimated_sale_price`, `estimated_gross_margin`, `market_readiness`, `target_market`, `expected_launch_year`, `tags`, `description` | `description_vector` | content=`description`, keywords=`trends_addressed`,`tags` |

- Embedding dimensions from `AZURE_OPENAI_EMBEDDING_DIMENSIONS` (default `1536`).
- HNSW profile named `hnsw`; one semantic configuration per index.
- Env var index names: `AZURE_SEARCH_PRODUCTS_INDEX_NAME`,
  `AZURE_SEARCH_PERSONAS_INDEX_NAME`, `AZURE_SEARCH_INNOVATIONS_INDEX_NAME`.

### 2.2 Ingestion

**Reference:** `scripts-samples/ingest_knowledge.py` (`_Embedder` class, Azure OpenAI
embeddings via `AZURE_OPENAI_EMBEDDING_DEPLOYMENT_NAME`, batched `upload_documents`,
`DefaultAzureCredential` fallback).

- Read the three JSON files directly (no markdown parsing needed — descriptions already
  written). Map each record → search document, embedding the `description` text into
  `description_vector`.
- Batch uploads (≈500 docs/batch); embeddings optional (skip vectors if
  `AZURE_OPENAI_ENDPOINT` unset — text/semantic search still works).

## 3. MCP Servers (Azure Container Apps)

All servers: **FastMCP** streamable‑HTTP transport, `/health` route, `_build_auth()`
using `RemoteAuthProvider` + `AzureJWTVerifier`, gated by `ENTRA_AUTH_ENABLED`
(+ `MCP_AUTH_CLIENT_ID`, `AZURE_TENANT_ID`, `MCP_PUBLIC_BASE_URL`). Each publishes
telemetry to **Application Insights** via `azure-monitor-opentelemetry`
(`configure_azure_monitor` on `APPLICATIONINSIGHTS_CONNECTION_STRING`, added to `main()`).
Each ships a `Dockerfile`, `requirements.txt`, `tools.json`, and
`register-external-mcp-server.json`.

**Reference for the auth block, health route, `main()`:** `src-samples/finance_mcp_server/server.py`
and `src-samples/product_data_mcp_server/server.py`.

### 3.1 `product_mcp_server` (AI Search backed)
**Reference:** `src-samples/product_data_mcp_server/` (structure, `tools.json`, Dockerfile,
`register-external-mcp-server.json`) + AI Search query pattern from
`src-samples/employee_advisory_agent/agent.py` (`search_financial_products`,
`SearchClient`, OData filter, `select`, `top`).
- Port `8093`. Env `PRODUCT_MCP_HOST/PORT`, `AZURE_SEARCH_ENDPOINT`, index name.
- **Tools:** `list_products(category?, market?, brand?, competitor_only?)`,
  `get_product(product_id)`, `search_products(query, top)` (semantic/vector/hybrid over
  `description`), `list_categories()`, `list_brands()`, `get_positioning(product_id)`.

### 3.2 `persona_mcp_server` (AI Search backed)
**Reference:** same server skeleton as 3.1; search pattern from the agent search tool.
- Port `8094`. Env `PERSONA_MCP_HOST/PORT`, `AZURE_SEARCH_ENDPOINT`, index name.
- **Tools:** `search_personas(query, top)` (vector/semantic over backstory `description`),
  `list_personas(market?, category?, age_group?, income_band?)`,
  `get_persona(persona_id)`, `find_personas_by_interest(interest)`.

### 3.3 `research_mcp_server` (AI Search backed)
**Reference:** same server skeleton; search pattern from the agent search tool.
- Port `8095`. Env `RESEARCH_MCP_HOST/PORT`, `AZURE_SEARCH_ENDPOINT`, index name.
- **Tools:** `search_innovations(query, top)`, `list_innovations(category?, target_market?,
  market_readiness?, max_launch_year?)`, `get_innovation(innovation_id)`,
  `list_trends()` (distinct `trends_addressed`).

### 3.4 `market_insights_server` (static file backed)
**Reference:** `src-samples/finance_mcp_server/server.py` (static/in‑process server, no
external data store, computes over loaded data). Load `data/market_sales_monthly.json` and
`data/market_sales_annual.json` at startup (`lru_cache`), similar to how
`product_data_mcp_server` loads its JSON catalogue.
- Port `8096`. Env `MARKET_MCP_HOST/PORT`, `MARKETING_DATA_DIR` (default `/app/data`).
- **Tools:** `get_sales(market?, category?, product_id?, year?)`,
  `get_market_share(market, category?, year?)`, `get_growth_rates(market?, category?)`,
  `compare_margins(category, markets[])`, `get_forecast(market, category?)`,
  `competitor_performance(market?, brand?)`.

### 3.5 Per‑server descriptor files
- `tools.json` — machine‑readable tool catalogue (**Reference:**
  `src-samples/product_data_mcp_server/tools.json`).
- `register-external-mcp-server.json` — Foundry registration descriptor with
  `authType: EntraID`/`none` (**Reference:**
  `src-samples/product_data_mcp_server/register-external-mcp-server.json`).
- `Dockerfile` — Python 3.13‑slim, copy `src` + `data`, expose port, run module
  (**Reference:** `src-samples/product_data_mcp_server/Dockerfile`).

## 6. Identity & Authentication

**Reference:** `scripts-samples/auth_helpers.py` (`entra_auth_enabled`,
`ensure_mcp_app_registration`, `grant_mcp_role_to_principal`, `Mcp.Invoke` role,
`resolve_mcp_audience`, `delete_mcp_app_registration`),
`scripts-samples/create_mcp_agent_identity_connections.py` (agent‑identity `remote-tool`
connections), `scripts-samples/grant_agent_identity_mcp_role.py` (grant `Mcp.Invoke` to
the agent's Entra Agent Identity via Graph).

- One Entra app registration per MCP server (`<app>-mcp-auth`, `api://<appId>`,
  `Mcp.Invoke` app role). Deploy scripts inject `ENTRA_AUTH_ENABLED`, `MCP_AUTH_CLIENT_ID`,
  `AZURE_TENANT_ID`, `MCP_PUBLIC_BASE_URL` into each container app.
- For each toolbox/agent, create an agent‑identity connection to each MCP server and grant
  the agent identity the `Mcp.Invoke` role.
- `ENTRA_AUTH_ENABLED=false` turns auth off everywhere (servers run anonymous, toolboxes
  registered without `project_connection_id`).
